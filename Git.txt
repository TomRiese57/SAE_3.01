git add : permet d'indexer les fichiers ou dossiers afins qu'ils soient suivis dans le git.
git clone https://gitlab.univ-lorraine.fr/riese3u/tp-git.git : cette commande me permet de copier le dépôt distant sur mon dépôt local.
git commit -m : permet de créer un message de sauvegarde des fichiers suivis par git.
git commit -a -m : me permet de créer un message de sauvegarde avec la sauvegarde des fichiers déjà suivis.
git init : permet de créer un répertoire .git sur le répertoire où l'on se trouve.
git log : cette commande met permet de voir les anciens commits présent sur le dépôt.
git merge origin/main : cette commande me permet de résoudre des conflits entre les dépôts distant et locaux qui seraient différents.
git pull : me permet de mettre à jour mon dépôt local avec le dépôt distant.
git push origin --tags : cette commande me permet d'envoyer mes tags du dépôt local au dépôt distant.
git push --et-upstream origin master : cette commande me permet d'envoyer tous les fichiers présent sur mon dépôt local .git local sur le dépôt distant gitLab.
git remote -v : cette commande me permet de vérifier si le dépôt distant sur gitLab est bien lier avec le depot local.
git remote add origin https://gitlab.univ-lorraine.fr/riese3u/typescript_riese_tom.git : cette commande me permet de lier mon dépôt .git local à mon dépôt distant gitLab.
git status : cette commande me donne tous les répertoires et fichiers qui ne sont pas suivis dans le .git.
git tag : cette commande me permet de faire un nouveau tag sur le dépôts local
touch .gitignore : permet de stocker tous les fichiers ou dossiers qui ne doivent pas être considérés par git.
